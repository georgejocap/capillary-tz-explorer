Capillary Platform
Timezone reference for every module
The same date can look different in InTouch versus the API — and the behavior varies across modules. This guide explains why, so your team stops chasing timezone bugs.
Org Timezone
Your company's clock
Set in Organization Settings. Every InTouch screen — promotions, members, journeys — shows dates in this timezone. When your team says "9 AM", this is whose 9 AM they mean.
Asia/Kolkata · Asia/Singapore · UTC
Server Timezone
What the API sends back
Capillary servers run in data centers with their own clocks. Every API response uses the server's timezone — regardless of your org's timezone. India cluster = IST, EU = UTC.
India +05:30 · EU UTC · APAC2 +08:00 · US ~−05:00
Input Timezone
Timezone you include when sending
When sending a date to Capillary via API, include your timezone (+05:30 for IST, Z for UTC). Without it, Capillary can't tell which 9 AM you mean — and promotions can fire at the wrong time.
2026-01-23T09:00:00+05:30 → IST stored ✓
The key thing to know before you start

InTouch and the API show the same event at different times — and that's by design.

InTouch (what your team sees)
23 Jan 2026 · 09:00 IST
Converted to org timezone for display
≠
API response (what a developer gets)
2026-01-23T03:30:00+00:00
Server timezone — same moment, different clock

Both are correct — they represent the same point in time. The modules below show which timezone each feature uses and where the gaps are.

Set Org Timezone in InTouch
1
Click the Profile icon (top right) → Organization Settings
2
Go to Organization Profile → click Edit
3
Click the "Set Org TimeZone" tab
4
In Time Zone, add all timezones your org operates in
5
In Base Time Zone, select your HQ timezone — default across all UI modules
6
Click Submit. All UI modules will now show dates in your base timezone.
